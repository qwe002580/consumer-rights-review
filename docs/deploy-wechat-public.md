# 公众号公网部署说明

这套站点已经整理成可部署版本，适合放到公众号菜单、自动回复或文章链接里打开。

## 推荐上线方式

推荐你用这一条：

1. 国内云服务器
2. 独立域名
3. HTTPS 证书
4. Docker 部署

这样做的原因：

- 微信里打开更自然
- 客户不需要和你的电脑在同一个 Wi-Fi
- 你关掉本地电脑，网站也还能继续用
- 后台案件数据会一直保留在服务器

## 你最后会得到什么

上线后你可以把下面这种地址直接放到公众号里：

- `https://你的域名/`

后台入口：

- `https://你的域名/cases`

## 部署前要准备

你需要准备 4 样东西：

1. 一台云服务器
2. 一个域名
3. HTTPS 证书
4. 下面这份生产环境配置

建议：

- 如果你希望中国大陆打开更稳，优先选国内云服务器
- 如果服务器放在中国内地，域名通常也需要备案
- 如果你暂时不想备案，可以先放中国香港节点，但中国内地访问速度和稳定性通常会弱一些

上面这两条是基于国内常见部署规则的经验判断。

## 生产环境变量

把 `.env.production.example` 复制成服务器上的 `.env.production`，再填写真实内容：

```env
DATABASE_URL="file:./data/dev.db"
DEEPSEEK_API_KEY="你的 DeepSeek Key"
DEEPSEEK_API_BASE_URL="https://api.deepseek.com"
DEEPSEEK_MODEL="deepseek-v4-flash"
NEXT_PUBLIC_CONSULTATION_URL="你的企微获客链接"
```

## Docker 部署

### 1. 上传项目到服务器

把整个项目上传到服务器任意目录，例如：

```bash
/srv/consumer-rights-review
```

### 2. 构建镜像

```bash
docker build -t consumer-rights-review .
```

### 3. 启动服务

```bash
docker run -d \
  --name consumer-rights-review \
  -p 3000:3000 \
  --restart unless-stopped \
  --env-file .env.production \
  -v $(pwd)/data:/app/data \
  consumer-rights-review
```

说明：

- 网站会跑在服务器 `3000` 端口
- 数据库会放在当前目录的 `data` 文件夹
- 即使你重启服务器，案件数据也还在

## 域名和 HTTPS

建议你再做一层反向代理，把 `https://你的域名` 转到本地 `3000` 端口。

你可以用：

- 宝塔面板
- 1Panel
- Nginx
- Caddy

最省事的做法：

1. 域名解析到服务器 IP
2. 在面板里新建站点
3. 反向代理到 `http://127.0.0.1:3000`
4. 申请 HTTPS 证书
5. 开启强制 HTTPS

## 给公众号使用

上线后你可以把下面任意地址放进公众号：

- 菜单跳转链接
- 自动回复里的链接
- 文章中的原文链接
- 企微欢迎语里的链接

推荐对客户公开的入口：

- `https://你的域名/`

后台入口不要直接公开给客户：

- `https://你的域名/cases`
