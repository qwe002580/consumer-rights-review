# 企业微信新案件通知设计

## 目标

客户提交退款纠纷自测并成功保存案件后，系统立即向指定企业微信群发送一条新案件通知，帮助管理员及时进入后台处理。

## 通知方式

使用企业微信群机器人 Webhook。Webhook 地址只通过服务端环境变量 `WECOM_CASE_WEBHOOK_URL` 读取，不写入源代码、前端资源、日志或 Git 仓库。

## 数据流程

1. 客户提交资料，服务端完成格式校验和案件分析。
2. 服务端将案件保存到数据库。
3. 保存成功后，服务端调用企业微信机器人 Webhook。
4. 无论通知是否发送成功，客户都会收到已生成的分析结果。

通知属于附加能力，不得阻塞或破坏案件主流程。请求设置短超时；企业微信不可用、返回错误或未配置 Webhook 时，仅记录不含密钥和客户隐私的服务端错误。

## 通知内容

消息使用企业微信 Markdown 格式，包含：

- 标题：新退款纠纷案件
- 案件编号
- 纠纷类型
- 支付金额
- 当前处理进度
- 处理优先级
- 提交时间
- 案件后台详情链接

消息不得包含客户姓名、手机号、微信号、补充说明、聊天内容或其他原始材料。后台详情链接使用 `PUBLIC_SITE_URL` 拼接案件编号；未配置时使用当前请求来源作为后备地址。

## 模块边界

新增独立服务端通知模块，负责：

- 读取并校验通知配置
- 生成不含隐私的消息内容
- 调用企业微信机器人接口
- 将发送结果转换为成功、跳过或失败状态

案件提交接口只负责在数据库保存成功后调用该模块，不直接拼装企业微信请求。

## 错误处理

- 未配置 Webhook：跳过通知，不影响案件提交。
- 网络超时或企业微信接口错误：记录简短错误，不包含完整 Webhook 地址。
- 企业微信返回非成功状态：按失败处理，不向客户暴露内部错误。
- 案件保存失败：不发送通知，继续返回现有保存失败响应。

## 测试与验收

- 单元测试验证通知内容包含案件摘要和后台链接。
- 单元测试验证通知内容不包含姓名与联系方式。
- 单元测试验证未配置、网络失败和企业微信错误均不影响案件提交。
- 路由测试验证仅在案件保存成功后触发通知。
- 线上配置完成后发送一条不含真实客户资料的测试通知。
- 最终通过真实客户后续提交验证通知能到达指定企业微信群。

## 部署配置

Zeabur 新增以下服务端环境变量：

- `WECOM_CASE_WEBHOOK_URL`：企业微信群机器人 Webhook
- `PUBLIC_SITE_URL=https://consumer-rights-review.zeabur.app`

环境变量更新后重新部署服务。机器人密钥不得出现在提交记录、构建输出或公开页面中。
