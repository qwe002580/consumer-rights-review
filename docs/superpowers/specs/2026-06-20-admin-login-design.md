# 后台登录保护设计

## 目标

为案件管理后台增加服务端登录保护。管理员使用环境变量中配置的账号和密码登录，成功后保持登录 7 天。客户评估页面和提交接口保持公开。

## 访问范围

- `/admin/login`：公开的后台登录页。
- `/cases`：仅登录管理员可访问。
- `/cases/[id]`：仅登录管理员可访问。
- `PATCH /api/cases/[id]`：仅登录管理员可调用。
- `POST /api/analyze`：保持公开，供客户提交评估资料。

未登录访问后台页面时跳转到 `/admin/login`。未登录调用受保护接口时返回 `401`，不返回案件内容。

## 登录与会话

- 管理员账号通过 `ADMIN_USERNAME` 配置，线上值为 `admin`。
- 管理员密码通过 `ADMIN_PASSWORD` 配置，不写入前端代码或 Git 仓库。
- `ADMIN_SESSION_SECRET` 用于签名会话令牌，部署时生成随机值。
- 登录接口在服务端校验账号密码，并采用恒定时间比较降低侧信道风险。
- 登录成功后设置 `HttpOnly`、`Secure`、`SameSite=Lax` Cookie，有效期 7 天。
- 会话令牌包含到期时间并带服务端签名，篡改或过期后均视为未登录。
- 退出登录会清除 Cookie，并返回登录页。

## 页面体验

登录页沿用当前米白与深绿色视觉风格，提供账号、密码、登录按钮和清晰的错误提示。后台列表顶部增加“退出登录”入口。已登录访问登录页时直接进入案件列表。

## 错误处理

- 账号或密码错误：统一提示“账号或密码不正确”，不透露具体哪项错误。
- 缺少服务器安全变量：登录失败并记录服务端配置错误，不向浏览器暴露秘密。
- 会话无效或过期：清除访问权限并要求重新登录。

## 测试与验收

- 正确凭据可创建 7 天有效的签名会话。
- 错误凭据不能创建会话。
- 篡改或过期会话不能访问后台。
- 未登录访问案件列表、详情会跳转登录页。
- 未登录修改案件接口返回 `401`。
- 登录后可访问案件列表、详情并正常保存状态。
- 退出后再次访问后台会被要求登录。
- 正式构建、现有测试和线上移动端页面检查全部通过。
